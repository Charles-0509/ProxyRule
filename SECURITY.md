# 安全模型

## 信任边界

- 正式配置只从 `Charles-0509/ProxyRule` 的 HTTPS 地址下载规则，不经过
  第三方 GitHub 代理站。
- 规则更新通过 OpenClash 的 `规则更新` 代理组访问 GitHub，因此代理节点
  只承载 TLS 隧道，不能正常替换 GitHub 返回的数据。
- `main` 只包含人工审核后合并的规则；定时任务只能创建 PR。
- Release 标签不可变，并提供 `SHA256SUMS` 供离线校验和回滚。

## 不要提交的内容

- 机场订阅 URL、转换服务 URL、Token、Cookie、控制器密码和节点凭据。
- OpenClash 运行日志、缓存 provider 文件或包含真实公网地址的故障包。

## 路由器边界

- 默认控制器只监听 `127.0.0.1:9090`。
- `allow-lan: true` 是软路由透明代理所需，但 OpenWrt 防火墙必须禁止从
  WAN 访问 7890、7892、7895、7874 和 9090。
- 如果主动把控制器开放到 LAN，必须设置每台路由器独有的长随机密码。

安全问题请通过不包含敏感信息的 GitHub Issue 报告；涉及凭据时应先撤销
凭据，再发布经过清理的最小复现信息。
