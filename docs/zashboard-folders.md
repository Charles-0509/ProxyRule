# 导入 Zashboard 文件夹布局

新版 Zashboard 的“策略组”和“节点组”属于浏览器本地设置，不是 Mihomo
配置文件的一部分。仓库提供了 [Zashboard 设置文件](../config/zashboard-settings.json)，
用于建立以下三个标签：

- **策略组**：业务分流策略，顶部是“手动选择”。
- **故障转移**：所有地区和全节点的“故转”组。
- **节点组**：所有手动节点组和自动测速组。

## 导入方法

1. 在 Zashboard 中打开“设置”。
2. 找到“导入设置”，选择下载后的 `zashboard-settings.json`。
3. 确认覆盖 `config/proxy-folders`、`config/proxy-folder-mode-setting`、
   `config/speedtest-mode` 和 `config/speedtest-timeout`。
4. 页面刷新后回到“代理”页面。

导入文件还会把测速方式设为“核心控制”，超时设为 8 秒。这样 Selector、
URL-Test 和 Fallback 组都由 Mihomo 的组测速接口完成，避免浏览器逐个请求节点
导致假超时。

Zashboard 将这些设置保存在当前浏览器中。更换电脑、浏览器配置文件或清理站点
数据后，需要重新导入一次。

## 新故障转移结构

每个地区只包含三个组：

- `美国-手动` / `美国-自动`：直接容纳美国节点。
- `美国-故转`：依次检测“美国-手动”“美国-自动”“所有-自动”。

`手动选择` 用于选择希望优先使用的地区。例如选中“香港-故转”时，它会
优先使用香港手动组当前节点；该节点失效时尝试香港自动组，香港节点全部失败
后改用“所有-自动”选择其他可用节点。

`fallback` 组由 Mihomo 根据健康检查自动决定当前成员，本身不能像 `select`
一样强制选择成员。需要指定某个美国节点时，应在“美国-手动”中选择节点，
再让“美国-故转”负责健康检查和自动接管。
